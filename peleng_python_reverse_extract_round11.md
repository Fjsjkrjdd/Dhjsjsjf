## 1) Packed-state extractor
`FUN_00413A75()` and `FUN_00414704()` are the same packed-state extractor family:
```c
if ((ctx[400] & 1) == 0) {
    return ((ctx[0x101] & 2) != 0);
}
switch (ctx[400] & 6) {
case 0: return 4;
case 2: return 2;
case 4: return 5;
case 6: return 3;
default: return 0xff;
}
```

### Bits
- `ctx+400 bit0` selects mode.
- `ctx+400 bits1..2` map through a LUT: `00→4`, `01→2`, `10→5`, `11→3`.
- `ctx+400 bit2` is also used as a standalone gate in `FUN_00413B1A()`.
- `ctx+0x101 bit1` is used when `ctx+400 bit0 == 0`.
- `FUN_00413B4D()` checks `READ16(ctx,0x4B) != 0`.

## 4) Practical interpretation
- `лев / прав / обе` are stored as lookup strings in `.data`.
- `открытая / с кольцами / с буксой` are stored as lookup strings in `.data`.
- `есть / нет` is a separate bool-style lookup.
- The defect catalog around `0x501998..0x501e1c` is a long string table that looks like the final human-readable defect taxonomy.
- `FUN_00414B68()` routes numeric codes into the larger descriptor families that eventually point into these string blocks.
